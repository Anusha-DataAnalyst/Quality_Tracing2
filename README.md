# Quality_Tracing2
# AI Traceability Experiment (iTrust2)

## Project Overview
This project evaluates how AI-assisted coding (GitHub Copilot) generates traceability links between:
- Functional Requirements (FR)
- Non-Functional Requirements (NFR)
- Source Code (iTrust2 system)

## System Under Study
iTrust2 healthcare system (Spring MVC project)

## Research Questions
RQ1: How can AI agents improve software self-documentation?
RQ2: How can traceability quality be improved using structured prompts?

## Method
- Define FR and NFR requirements
- Generate AI-assisted code using Copilot
- Extract trace links from generated code
- Evaluate correctness manually

## Output
- Requirement documents
- Trace mappings
- Evaluation results
